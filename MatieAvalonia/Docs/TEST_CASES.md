# Тестирование (блок 5 ТЗ)

## 1. Unit-тесты: время последнего изменения (`DbTimestamp`)

Единственная выделенная логика меток `Updated_At` / «наивного» времени для PostgreSQL — класс `MatieAvalonia.Classes.DbTimestamp` (`Now`, подмена `LocalClock`, сброс `UseSystemLocalClock`).

Запуск: `dotnet test MatieAvalonia.sln`.

| № | Имя теста | Проверяемое поведение |
|---|-----------|------------------------|
| U1 | `Now_Kind_IsUnspecified` | `Now` всегда с `DateTimeKind.Unspecified` (ожидание `timestamp without time zone`) |
| U2 | `Now_PreservesComponents_FromLocalClock` | Компоненты даты/времени совпадают с подставленными часами |
| U3 | `Now_DoesNotReturnUtcKind` | Даже если часы отдают UTC, результат не остаётся в Kind UTC |
| U4 | `UseSystemLocalClock_AfterOverride_UsesRealClock` | После сброса используется реальное локальное время |
| U5 | `Now_PreservesMilliseconds_ForLastModifiedStamp` | Доли секунды сохраняются (точность метки изменения) |

| Набор | Количество | Статус (обновлять после прогона) |
|-------|------------|-------------------------------------|
| `DbTimestampTests` | 5 | Пройдено |
| `BookingScheduleRulesTests` | 6 | Пройдено |
| **Всего** | **11** | Пройдено |

## 2. Ручное тестирование: 5 тест-кейсов (по одному на функцию)

Один тест-кейс на каждую из пяти ключевых функций приложения. Заполните **фактический результат** и **комментарии** после прогона.

| № | Название теста | Проверяемая функция | Шаги для получения результата | Ожидаемый результат | Фактический результат | Комментарии |
|---|----------------|---------------------|-------------------------------|---------------------|------------------------|-------------|
| 1 | Вход в систему | Авторизация (`AuthPage`) | Открыть приложение; ввести существующий логин и пароль; нажать «Войти». | Открывается главное окно, в шапке отображается ФИО пользователя. | | |
| 2 | Просмотр каталога | Каталог услуг (`ServicesCatalogPage`) | Войти как клиент; открыть раздел каталога; при наличии данных — выбрать коллекцию в панели фильтра. | Отображается список услуг; при выборе коллекции список соответствует фильтру. | | |
| 3 | Запись на услугу | Создание записи (`BookingCreatePage`) | Выбрать услугу, мастера, дату/время в будущем; сохранить. | Запись создана; доступен список «Мои записи» с новой строкой (или переход к нему). | | |
| 4 | Пополнение баланса | Карта и баланс (`CardTopUpPage`) | Открыть пополнение; ввести корректные данные карты (или выбрать сохранённую) и сумму > 0; сохранить. | Баланс увеличен; карта отображается в списке при новой карте. | | |
| 5 | Сохранение услуги модератором | Обновление услуги и метка времени (`ModeratorServiceFormPage`) | Войти как модератор; открыть редактирование услуги; изменить, например, цену; сохранить; проверить поле «Обновлено». | Услуга сохранена; отображается актуальная дата/время обновления (метка `Updated_at` через `DbTimestamp`). | | |
